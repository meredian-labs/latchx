# Security Policy

Latch is early software. Treat reports as review aids, not security guarantees.

## Supported Versions

| Version | Supported |
| --- | --- |
| 0.1.x | Yes |

## Reporting A Security Issue

Do not open public issues for vulnerabilities.

For now, report security concerns privately to the maintainers. Include:

- affected package or command,
- `latchx` version,
- operating system and Node/npm versions,
- reproduction steps,
- any relevant audit report JSON,
- expected and actual behavior.

## Token Safety

Never paste npm tokens, GitHub tokens, cloud tokens, or private registry tokens into issues, pull requests, logs, or chat transcripts. If a token is exposed, revoke it immediately and create a replacement.
